# 알고리즘의 개요
알고리즘(Algorithmn)은 특정한 작업을 수행하는 명령이나 연산자의 집합을 의미합니다.
이러한 작업을 할 때 필요한 경우 외부에서 입력을 받고 작업을 수행한 후에 그 결과물을 출력하게 됩니다.

# 알고리즘의 특성
일반적으로 알고리즘은 다음과 같은 조건을 만족시켜야 합니다.
1. 입력(input) : 외부에서 제공되는 자료가 있을 수 있다.
2. 출력(output) : 적어도 한가지의 결과 자료가 나온다.
3. 명확성(definiteness) : 각 명령이나 연산자들은 모호하지 않고 명확해야 한다.
4. 유한성(finiteness) : 알고리즘의 명령대로 수행하면 한정된 단계를 처리하고 반드시 종료되어야 한다.
5. 효율성(effectiveness) : 모든 명령들은 종이에 기록하고 실행 과정을 예상할 수 있을 정도로 기본적이어야 한다.

# 알고리즘의 분석 기준
알고리즘의 분석은 다름과 같은 기준에 따라 이루어 집니다.
1. 시간 복잡도 : 알고리즘을 수행하는데 필요한 시간의 양을 나타낸다.
2. 공간 복잡도 : 알고리즘을 수행하는데 필요한 기억장소의 양을 나타낸다.
3. 정확성(correctness) : 알고리즘에 결함이 없이 예상한대로 잘 동작하는가의 여부와 입력에 대한 정확한 결과를 출력해 내는가를 나타낸다.
4. 최적성(optimality) : 알고리즘이 해결하고자 하는 문제를 최소의비용으로 해결하고 있는지를 나타낸다.
5. 단순성(simplicity) : 알고리즘의 전체적인 구조나 흐름이 읽고 이해하기 쉽고 수정이 용이한지를 나타낸다.
